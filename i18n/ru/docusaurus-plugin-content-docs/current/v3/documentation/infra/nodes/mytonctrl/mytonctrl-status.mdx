# MyTonCtrl status

Вот объяснение вывода команды `status`.

![status](/img/docs/mytonctrl/status.png)

## Состояние сети TON

### Имя сети

Возможные значения: `mainnet`, `testnet`, `unknown`. `unknown` не должен быть напечатан всегда.

### Количество валидаторов

Есть два значения: один зеленый и один желтый. Green is the number of online validators, yellow is a number of all
validators.
Должно быть целым числом, больше 0, MyTonCtrl получить его командой `getconfig 34`,
проверить [этот раздел (параметр 32-34-and-36)](/v3/documentation/network/configs/blockchain-configs#param-32-34-and-36) еще больше.

### Количество шардингов

Должно быть целым числом больше 0, имеет зеленый цвет.

### Количество предложений

Есть два значения: один зеленый и один желтый. Зеленый цвет - это количество новых предложений, желтый - количество "всех предложений".

### Количество жалоб

Есть два значения: один зеленый и один желтый. Green is a number of `new complaints`, yellow is a number
of `all complaints`.

### Статус выборов

Может быть зеленым текстом `open` или желтым `closed`.

## Статус локального валидатора

### Индекс проверки

Если индекс валидатора больше или равен 0 (и это должно быть так, если активен режим валидации), он должен быть зеленым, иначе - красным.

### ADNL адрес локального валидатора

Просто ADNL адрес.

### Локальный адрес кошелька валидатора

Адрес для разбивки должен быть действительным TON-адресом. 

### Баланс кошелька валидатора

Баланс кошелька.

### Средняя нагрузка

Имеет формат `[int]: int, int, int`. Первый — это количество процессоров, другие — среднее значение нагрузки системы за последние 1, 5 и 15 минут.

### Средняя нагрузка по сети

Три целых числа, то же самое логика, что и для `средняя нагрузка`: среднее значение нагрузки системы в течение последних 1, 5 и 15 минут.

### Загрузка памяти

Две пары целых чисел, абсолютное и относительное использование памяти и swap.

### Средняя нагрузка на диски

То же что и при загрузке памяти, но для всех дисков.

### Mytoncore status

Должно быть зеленым, рассказывает сколько времени Mytoncore работает.

### Статус локального валидатора

Должно быть зеленым, указывает, сколько времени работает локальный валидатор.

### Локальный валидатор не синхронизирован

Целое число должно быть меньше 20 (оно будет зеленое).

### Локальная сериализация последнего состояния валидатора

Показывает количество блоков вне службы мастер-цепочки

### Размер базы данных валидатора

Абсолютная нагрузка должна быть меньше 1000 GB, относительная нагрузка должна быть меньше 80%.

### Версия mytonctrl

Хэш коммита и название ветки.

### Проверка версии

Хэш коммита и название ветки.

## Конфигурация сети TON

### Адрес конфигуратора

Адрес конфигуратора, проверьте [этот параметр 0](/v3/documentation/network/configs/blockchain-configs#param-0) для получения дополнительной информации.

### Адрес электродвигателя

Адрес электродвигателя, проверьте [этот параметр 1](/v3/documentation/network/configs/blockchain-configs#param-1), чтобы узнать больше.

### Период проверки

Период проверки в секундах, проверьте [этот параметр 15](/v3/documentation/network/configs/blockchain-configs#param-15), чтобы узнать больше.

### Продолжительность выборов

Продолжительность выборов в секундах, проверьте [этот параметр 15](/v3/documentation/network/configs/blockchain-configs#param-15), чтобы узнать больше.

### Период ожидания

Удерживайте период в секундах, проверьте [этот параметр 15](/v3/documentation/network/configs/blockchain-configs#param-15), чтобы узнать больше.

### Минимальная цена

Минимальная ставка в TONs, проверьте [этот параметр 17](/v3/documentation/network/configs/blockchain-configs#param-17) для получения дополнительной информации.

### Максимум долей

Максимальное количество в TONs, проверьте [этот параметр 17](/v3/documentation/network/configs/blockchain-configs#param-17), чтобы узнать больше.

## TON timestamps

### Запуск сети TON

Время, когда была запущена текущая сеть (mainnet или testnet).

### Начало цикла проверки

Временная метка начала цикла проверки; она будет зеленая, если она представляет собой будущий момент.

### Конец цикла проверки

Временная метка окончания цикла проверки; она будет зеленая, если она представляет собой будущий момент.

### Начало выборов

Временная метка начала выборов; она будет зеленая, если она представляет собой будущий момент.

### Окончание выборов

Временная метка окончания выборов; она будет зеленая, если она представляет собой будущий момент.

### Начало следующих выборов

Временная метка начала следующих выборов; она будет зеленая, если она будет представлять будущий момент.
