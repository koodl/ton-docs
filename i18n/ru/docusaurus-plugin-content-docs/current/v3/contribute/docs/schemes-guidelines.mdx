# Руководства по графическим пояснениям

Важнейшее значение имеет поддержание последовательности в документации, и для этого был разработан конкретный стандарт визуализации процессов в смарт-контрактах.

## Заметка графического пояснения

### График обработки сообщений

Чтобы изобразить обработку сообщений, рекомендуется использовать графическое представление, похожее на диаграмму смарт-контракта, заполненную метками для транзакций и сообщений.

Если порядок транзакций не имеет значения, можно убрать их метки. Это упрощает диаграмму, что облегчает прочтение и понимание деталей, связанных с сообщениями и контрактами.

- Избегайте использования большого количества различных и ярких цветов.
- Используйте модификацию фигур, например, с пунктирной граничной линией.
- Для лучшего понимания различные транзакции могут быть отображены с разными стилями линий (сплошные и пунктирные).

### Форматы и цвета

#### Шрифты

- Семейство **Inter** шрифтов для всех текстов в диаграммах.

#### Цвета - Светлый режим

- Карандаш Рисунок (по умолчанию)

#### Цвета - Темный режим

- Font `#e3e3e3`
- Фон `#2328`
- Light Highlight(стрелки и схемы границ) `#058dd2`
- Темный Highlight(стрелки и схемы границ) `#0088cc`
- InnerBackGround(для вложенных блоков) `#333337`

#### Политика контроля версий

- Установка диаграмм в документации формата SVG для схем для обеспечения читаемости на различных устройствах.
- Сохраняйте исходные файлы в Git репозитории проекта в каталоге "/static/visio", что облегчает их изменение в будущем.

### Диаграмма последовательности

В случае сложных и повторяющихся схем коммуникации между 2-3 действующими лицами рекомендуется использовать диаграмму последовательности. Для сообщений используйте обозначение общей стрелки синхронного сообщения.
